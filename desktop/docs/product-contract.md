# Product Contract -- LAUNCHPAD Desktop

## Kernversprechen

1. Kind sieht Spiel.
2. Kind drueckt Start.
3. Spiel startet.
4. Eltern haben vorher kuratiert.

Jede Funktion, jedes UI-Element und jeder technische Pfad muss auf diese vier Saetze zurueckfuehrbar sein. Was keinen dieser Saetze unterstuetzt, gehoert nicht ins Produkt.

## Primaerer Nutzer

| Rolle | Kontext | Geraet / Eingabe |
|---|---|---|
| Kind | Couch, Wohnzimmer, allein oder mit Geschwistern | Controller (Xbox, PlayStation, Nintendo) |
| Elternteil | Setup / Kuration, gelegentlich am selben PC | Maus + Tastatur, PIN-geschuetzt |

## Hauptsysteme

| System | Zweck |
|---|---|
| Lokale Spielbibliothek | Alle installierten Spiele als visuelle Kacheln |
| Elterliche Freigabe | Kuratierung und Altersfreigabe durch Eltern |
| Kinderprofile | Mehrere Kinder mit eigener Auswahl |
| Controller-first UI | Steuerung primaer ueber Gamepad, Maus sekundaer |
| Launch-Adapter | Stabiler Start ueber Steam, EGS, GOG, lokale EXE |

## Abgrenzung

Dieses Dokument definiert *was* das Produkt verspricht. Technische Umsetzung folgt in separaten Gates. Was explizit ausgeschlossen ist, steht in `non-goals.md`.
