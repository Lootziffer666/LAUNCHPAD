# Non-Goals -- LAUNCHPAD Desktop

Dieses Dokument listet alles, was LAUNCHPAD Desktop bewusst *nicht* ist und *nicht* werden soll.

## Kein Store-Ersatz

LAUNCHPAD verkauft keine Spiele, bietet keine Storefronts an und verlinkt nicht auf Kaufseiten. Die Bibliothek zeigt nur lokal installierte, von Eltern freigegebene Titel.

## Keine Community-Features

Kein Chat, keine Freundeslisten, keine Online-Multiplayer-Koordination, keine Kommentare.

## Keine Bewertungsautomatik als Wahrheit

Altersempfehlungen (USK, PEGI, ESRB) werden als Hinweis angezeigt, niemals als automatische Freigabe-Entscheidung. Die Kurationsentscheidung liegt immer beim Elternteil.

## Keine riesige Game-Detailseite

Maximal: Cover, Name, kurze Info, Trailer-Link. Kein Metacritic, keine Reviews, keine Achievements, keine Screenshots-Galerie.

## Keine automatische paedagogische Entscheidung

Das System trifft keine Entscheidungen darueber, ob ein Spiel "gut" oder "schlecht" fuer ein Kind ist. Es stellt Informationen bereit; Eltern entscheiden.

## Keine offene Browser-/Store-Navigation fuer Kinder

Das Kind hat keinen Zugang zu URLs, Suchfeldern oder externen Stores. Die Kinderoberflaeche ist eine geschlossene Umgebung.

## Kein Dashboard

Keine Nutzungsstatistiken, Zeitdiagramme oder Aktivitaetsprotokolle als primaere UI. Parental Controls sind einfache Schalter, keine Analytics-Plattform.

## Keine dichten Panels

Die UI zeigt wenige, grosse Elemente. Keine Sidebars, Toolbars oder verschachtelten Menues in der Kinderansicht.

## Kein Store-Noise

Keine Werbung, keine "Empfehlungen", keine "Was ist neu"-Banner, keine Benachrichtigungs-Badges.

## Keine winzigen Desktop-Controls als Hauptbedienung

Dropdown-Menues, kleine Checkboxen und Texteingaben gehoeren in die Elternansicht. Die Kinderansicht ist Controller-first mit grossen Touch-/Focus-Targets.
